# Milestones

## M0 — Documentation accepted

Deliverables:

- Product requirements.
- Architecture.
- Database design.
- Permission matrix.
- UI/UX direction.
- Implementation phases.

Exit criteria:

- No major unknowns around core workflow, users, or data model.

## M1 — App skeleton

Deliverables:

- Next.js app.
- Tailwind/shadcn.
- Arabic RTL shell.
- Supabase configured.

Exit criteria:

- App runs locally.
- Login page and dashboard shell exist.

## M2 — Secure database foundation

Deliverables:

- Tables.
- Indexes.
- RLS.
- Seed data.

Exit criteria:

- Cross-company access is blocked in tests.

## M3 — Auth/roles

Deliverables:

- Login.
- Profiles.
- Role-based navigation.

Exit criteria:

- Each role reaches correct dashboard.

## M4 — Items

Deliverables:

- Item list/search/create/edit.

Exit criteria:

- Admin can manage item master data.

## M5 — Shortage workflow

Deliverables:

- Add request.
- Request list.
- Status transitions.
- History.

Exit criteria:

- Pharmacist-sales rep workflow works end to end.

## M6 — Launch readiness

Deliverables:

- Tests.
- Security review.
- Deployment.
- Device QA.

Exit criteria:

- Staging production candidate passes checklist.
