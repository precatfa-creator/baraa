# User Flows

## Flow 1 — Pharmacist adds shortage

1. Pharmacist logs in.
2. Opens النواقص.
3. Taps إضافة ناقص.
4. Searches item by name/barcode.
5. Enters quantity and optional notes.
6. Saves.
7. Request appears with status ناقص.

## Flow 2 — Sales rep starts purchasing

1. Sales rep logs in.
2. Sees assigned active requests.
3. Opens a missing request.
4. Taps بدء الشراء.
5. Status changes to قيد الشراء.
6. History records user/time.

## Flow 3 — Sales rep fulfills request

1. Sales rep opens in-purchase request.
2. Taps تم التوفير.
3. Optional note.
4. Status changes to تم توفيره.
5. Pharmacist can see update.

## Flow 4 — Company admin manages assignment

1. Admin opens المستخدمون/الصيدليات.
2. Selects sales rep.
3. Assigns pharmacy.
4. Sales rep can now see pharmacy requests.
